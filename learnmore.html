So, you are interested to find out what's hiding behind the web site, which kind of technologies were used to build up these pages, what processes and procedures were used to manage and maintain the solution and much more? So let's dive in under the hood and have a fun to discover those secrets.
First of all you mustn't realize that procedure which were used here in the project cannot fit as a generic solution for another project, you may be inspire by it but every project is unique and have a differences which could be against my solution.

Architecture

At first for the seek of simplicity were used VMMV javascript framework Knockout to have simple VM control and basic data binding. But over time it turns out that such as simple library isn't as progressive as chosen framework which is Angular. Thanks of many desired features and excellent modularity it won my private competition.

So first things first

Because there isn't any need to have a dynamic data consumed from beck-end site I have decided to have a simple javascript object literal as a data source loaded asynchronously. Once there is solution for back-end let's start with the front-end solution which is a little bit progressive. I will make a short list of all front-end technologies and processes to have overview about the solution, but some not so interesting techniques will be omitted.

  • Design

    At first I've created several wireframes and layouts and as a result I used those materials as a inspiration for static layout created in graphics application.

  • Project Bootstrap

    In order to setup the project a several tools were used to have initial process as simple as possible. Here is the short example of used tools:

    • Bower packaging tools to boilerplate or initial setup of vendor libraries e.g. jQuery, angular, twiter bootstrap.
    • Grunt tasker tools to be able to clen, concat, deploy, uglify and execute those steps to managing project. Additional packages for Gurunt where were useful:
      • grunt-processhtml
      • grunt-contrib-uglify
      • grunt-contrib-clean
      • grunt-contrib-concat
      • grunt-contrib-copy
      • and much more

  • CSS styling

    Preprocessor sass together with compass where useful to boost performance and efficiency of page styling. I have also take advantage of source map in order to integrate with chrome developer tools to have ability watch live changes of .sass files and transform it to css files on the fly.

  • Angular as a core framework has advantage to use all common techniques a processes like controllers, directives, services to have excellent logic and file organization over a project. Integration with extensions:

    • angular-bootstrap
    • angular-resource

  • Build process

    Finally here is describe a little bit more about final process of deployment to production environment. Also a few future improvement steps which are considered to be important to a project.

    Development to production envirinment consist of several steps before a folder package can be transported to the server. I'll descripe some of them:

    • clean up old package to be certain all previous files are wiped out
    • upload all necessary files e.g. images, fonts etc. to final package
    • merge css,js into one single file in order to minimize server requests
    • alter html files to respect server restriction
    • decrease main javascript and css file size by minifying process

    After those steps are the folder package ready to go to production environment.

    A one word about improvement process. I am going to analyze and identify the solution to expose critical path of resources and alter solution to be able to have page load under 1000 ms for best UX on mobile devices. Also page preload and asynchronous content consuming will be implement.

  • Final words

    The overall process may seems to be complicated but thanks the process of automations is any even small changes of project easy to be reflected to production environment. And even more there is almost eliminated possibility of error or forgotten step in development process.
